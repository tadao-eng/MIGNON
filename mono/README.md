# MONO — ミニマリズム持ち物管理 PWA

写真またはバーコードから持ち物を登録し、「今何を持っているか」「増えすぎていないか」を
ひと目で把握するための、自分専用・スマホ向け PWA です。

フレームワーク・ビルド工程なしの素の HTML/CSS/JS で、この `mono/` ディレクトリだけで完結します。

## 機能

- **登録** — カメラ撮影(`<input capture>`)+ 名前 / カテゴリ / 取得日 / バーコード / メモ
- **バーコード読取** — ブラウザ標準の `BarcodeDetector` API を優先、非対応環境(iOS Safari 等)では
  zxing-js に自動フォールバック。読取後は Open Food Facts で商品名の自動補完を試みる(失敗時は手動入力)
- **一覧** — 写真グリッド / リスト切替、カテゴリチップで絞り込み、取得日・名前・カテゴリで並び替え
- **増減サマリー** — ヘッダーに「今週 +n / −n」を常時表示
- **手放す** — 詳細からワンタップで除外し、方法(処分 / 譲渡 / 売却)と理由を履歴に記録。
  履歴からは「戻す」「完全削除」が可能
- **分析** — 現在の総数 / 今週の増減 / 手放した累計、総アイテム数の12週推移グラフ(SVG)、
  カテゴリ内訳バー(最多カテゴリの偏りをハイライト)
- **オフライン** — Service Worker がアプリシェルをキャッシュ。データは IndexedDB(写真は縮小 JPEG の Blob)に保存

## スマホへのインストール

このリポジトリは GitHub Pages で公開されているため、main にマージされると
**https://tadao-eng.github.io/MIGNON/mono/** で利用できます。

1. スマホのブラウザで上記URLを開く
2. - **Android(Chrome等)**: 画面上部に表示される「インストール」ボタンをタップ
     (表示されない場合はメニュー →「アプリをインストール」)
   - **iPhone / iPad(Safari)**: 共有ボタン →「ホーム画面に追加」
3. ホーム画面のアイコンから起動すると、アドレスバーのない全画面アプリとして動作します

一度開けばアプリシェルはキャッシュされるため、機内モードでも一覧・履歴・分析を閲覧できます。
データは端末内(IndexedDB)にのみ保存され、サーバには送信されません。

## ローカルでの動作確認

カメラと Service Worker は **HTTPS または localhost** でのみ動作します。`file://` では動きません。

```sh
cd mono
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

スマホ実機で試す場合は HTTPS が必要です。GitHub Pages に置くか、開発中なら
`npx serve` + トンネル(`ngrok http 8000` 等)や、同一LANなら Chrome の
`chrome://flags/#unsafely-treat-insecure-origin-as-secure` で開発機の IP を許可する方法があります。

ホーム画面への追加: モバイルブラウザでページを開き「ホーム画面に追加」を選ぶと
スタンドアロンのアプリとして起動します。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | 画面(一覧 / 追加 / 履歴 / 分析 + 各ダイアログ) |
| `style.css` | モノトーン + アクセント1色のミニマルデザイン |
| `app.js` | 画面制御・状態管理・描画 |
| `db.js` | IndexedDB ラッパー(`items` ストア1本、`status` で現役/手放し済みを区別) |
| `scanner.js` | バーコード読取(BarcodeDetector → zxing-js フォールバック) |
| `ai.js` | 品目情報の自動推定の拡張ポイント + バーコード商品名検索 |
| `sw.js` | Service Worker(アプリシェルのキャッシュ) |
| `manifest.json` | PWA マニフェスト |

## 将来の AI 連携について

画像からの品目名・カテゴリ自動推定は `ai.js` の `suggestItemInfo(photoBlob)` に隔離してあります。
現在は `null` を返すだけですが、ここを Vision 系 API(Claude API 等)の呼び出しに差し替えるだけで、
撮影直後にフォームへ自動入力される動線(`app.js` 側は実装済み)がそのまま有効になります。

データ移行が必要になった場合は `db.js` の `DB_VERSION` を上げ、`onupgradeneeded` で対応してください。
クラウド同期を足す場合も、ストレージ層が `db.js` に閉じているため差し替えは局所的です。
