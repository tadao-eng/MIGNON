# 中継サーバの設置手順(Cloudflare Workers)

MONO の AI判別は、Gemini APIキーをブラウザに直接保存する方式(手軽・キーは端末内のみ)と、
この中継サーバを使う方式(推奨・キーを端末にすら置かない)のどちらかを選べます。

中継サーバを使うと、Gemini APIキーはこの Worker にだけ保存され、スマホやPCのブラウザには
一切保存されません。写真は MONO からこの Worker を経由して Gemini に送られ、判定結果だけが
返ってきます。

以下はスマホのブラウザだけで完了する手順です(PCでも同じ流れで操作できます)。

## 1. Cloudflare の無料アカウントを作る

1. ブラウザで `dash.cloudflare.com` を開く
2. 「Sign up」からメールアドレスとパスワードを入力してアカウントを作成する(無料プランでOK)
3. 届いた確認メールのリンクをタップして認証する

## 2. Worker を作成する

1. Cloudflare ダッシュボードの左メニューから「Workers & Pages」を開く
2. 「Create」→「Workers」→「Create Worker」と進む
3. 名前を付ける(例: `mono-ai`)。この名前は後で出てくるURLの一部になります
4. 「Deploy」をタップし、いったんデフォルトのサンプルコードのままデプロイする

## 3. コードを差し替える

1. デプロイ後の画面から「Edit code」(コードを編集)を開く
2. エディタの中身を全選択して削除し、このリポジトリの `mono/proxy/worker.js` の中身を
   全部コピーして貼り付ける
3. 右上の「Deploy」をタップして保存・反映する

## 4. Gemini APIキーを Secret として登録する

1. Worker の管理画面で「Settings」(設定)タブを開く
2. 「Variables and Secrets」(変数とシークレット)を開く
3. 「Add」(追加)から、種別を **Secret** にして以下を入力する
   - 変数名: `GEMINI_API_KEY`
   - 値: 自分の Gemini APIキー(`aistudio.google.com/apikey` で無料発行できます)
4. 保存する(Secret は暗号化され、画面上にも再表示されません)

これでキーは Worker の中だけに存在し、MONO のブラウザ画面やこのリポジトリのコードには
一切含まれません。

## 5. MONO に中継サーバのURLを設定する

1. Worker の管理画面上部に表示されている URL をコピーする
   (`https://mono-ai.〈自分のサブドメイン〉.workers.dev` のような形式です)
2. MONO を開き、「分析」タブ →「AI設定」まで下にスクロールする
3. 「中継サーバURL」欄にコピーしたURLを貼り付けて「保存」をタップする
4. 「中継サーバ使用中」と表示されれば設定完了です

## 6. 動作確認とキーの削除

1. 「追加」タブで写真を撮り(またはアルバムから選び)、「AI分析」ボタンを押す
2. 品目名・カテゴリが推定されれば成功です
3. もし以前にブラウザ側の「Gemini APIキー」欄に直接キーを保存していた場合、
   中継サーバURLを設定した以降はそちらが優先されて使われるため、
   ブラウザ側のキーは「削除」ボタンで消して問題ありません
   (むしろ端末にキーを残さないためにも削除を推奨します)

## 7. 任意の追加防御

- **Cloudflare の Rate Limiting ルール**: Worker の管理画面から「Security」→「WAF」または
  「Rate limiting rules」を開き、この Worker のパスに対して短時間の大量リクエストを
  制限するルールを追加できます(無料プランでも簡易な設定が可能です)
- **ALLOWED_ORIGINS の編集**: `worker.js` 冒頭の以下の行を、自分が実際に MONO を公開している
  オリジン(ドメイン)だけに絞ることで、他サイトからのアクセスを防げます。
  ```js
  const ALLOWED_ORIGINS = ['https://tadao-eng.github.io', 'http://localhost:8000'];
  ```
  編集後は「Edit code」から差し替えて再度「Deploy」してください

## 仕組みとセキュリティ上の利点

- Gemini APIキーは Worker の Secret としてのみ保存され、ブラウザ(端末)には一切保存されません。
  端末を紛失・共有しても、キーが漏れることはありません
- この Worker は「持ち物の写真1枚を判定して名前とカテゴリだけを返す」専用の窓口です。
  判定に使うプロンプトは Worker 内で固定されており、リクエスト側から任意の指示や別の用途への
  流用はできません(汎用的な Gemini 中継としては使えない設計です)
- Origin(送信元サイト)を確認し、許可リストにないサイトからのリクエストは拒否します
