# LINE AI秘書 連携ガイド

OPOYO Learning の「毎日の単語10個+文法1課」をLINEで配信し、反復テストで定着させるための設定資料。

## データの場所

| ファイル | 内容 | URL |
|---|---|---|
| `curriculum.json` | **日別カリキュラム**(毎日: 英語5語+イ語5語+文法1課、+先取り分`extra`。要点・例文入り。日数は全単語が一巡するまで自動延長) | `https://tadao-eng.github.io/MIGNON/learn/curriculum.json` |
| `words.json` | 全単語(英1088/イ1011、順次拡張)+週テーマ | `https://tadao-eng.github.io/MIGNON/learn/words.json` |

GitHub raw でも取得可: `https://raw.githubusercontent.com/tadao-eng/MIGNON/main/learn/<ファイル名>`

## curriculum.json の構造

```json
{
  "days": [
    {
      "day": 1, "week": 1, "theme": "あいさつ・自己紹介",
      "words": {
        "en": { "review": false, "list": [ { "word": "Good morning", "meaning": "おはようございます", "category": "あいさつ・基本" } ] },
        "id": { "review": false, "list": [ ... ] }
      },
      "grammar": {
        "lang": "en", "no": 1, "review": false,
        "title": "be動詞（am / is / are）",
        "point": "「AはBです」は be動詞で表します。…",
        "examples": [ { "text": "I am busy.", "jp": "私は忙しい" } ]
      }
    }
  ]
}
```

- ボット側で **開始日(start_date)を1つ保持**し、`経過日数+1 = day` で `days[]` を引く
- `review: true` はその教材が一巡して復習に回った印(出題文言を「復習」に変えるとよい)
- **`extra`(倍ペース用)**: 各日は `extra` に追加分(翌日分の先取り: 単語10語+文法1課)を持つ。
  ユーザーが「もっと」「今日は倍」と言ったら `extra` を配信する。先取りした翌日は同じ内容が
  通常枠で流れるが、それはそのまま復習になるので問題ない。90日はマイルストーンであり、
  カリキュラムは180日分ある(学習は90日で終わらない)

## 1日の配信フロー(推奨)

| 時刻 | 配信 |
|---|---|
| 朝 8:00 | **今日の10語**(day N の en 5語 + id 5語)。意味・変化形付きで一覧 |
| 昼 12:30 | **今日の文法1課**(title + point + examples) |
| 夕 18:00 | **ミニテスト5問**(今日の10語から。4択で出題→回答に正誤+解説) |
| 夜 21:00 | 「アプリで1時間の学習を」リマインド(https://tadao-eng.github.io/MIGNON/learn/) |
| 夜 22:00 | 当日の報告(アプリの「LINEに今日の報告」)が来ていなければ**催促** |

ユーザーがアプリから送る報告のフォーマット(パースする場合の参考):

```
📚 OPOYO Learning Day 12/90
タスク 4/4（単語42枚 文法✅ 読解✅ 音読✅）
🔥 12日連続
苦手: postpone / berapa / lurus
```

「苦手:」の単語は翌日のミニテストに混ぜると効果的。

## LLM型秘書への貼り付け用プロンプト

ボットがLLMベース(システムプロンプトを設定できるタイプ)の場合、以下を追記:

```
# 語学コーチの役割
あなたはユーザーの語学コーチも兼ねる。ユーザーは90日間で英語とインドネシア語の
日常会話習得を目指しており、1日最低10単語+文法1課を覚える。

- カリキュラム: https://tadao-eng.github.io/MIGNON/learn/curriculum.json
  開始日は {START_DATE}。今日が Day 何日目かを計算し、その日の educational content を使う。
- 「今日の単語」と言われたら: その日の英語5語+インドネシア語5語を、意味・変化形付きで見やすく出す。
- 「もっと」「倍」と言われたら: その日の extra(追加10語+文法1課)を出す。やる気を必ず褒める。
- 「テスト」と言われたら: 直近3日分の単語から4択クイズを1問ずつ、計10問出す。
  1問ごとに正誤判定と短い解説。最後にスコアと間違えた単語のまとめ。
- ユーザーから「📚 OPOYO Learning Day…」の報告が来たら: 内容を記録し、短く具体的に褒める。
  「苦手:」の単語があれば翌日のテストに必ず含める。
- 22時までに報告がない日は「今日の報告がまだです。ストリークが切れます🔥」と催促する。
- 英語・インドネシア語の質問には例文つきで答え、新しく教えた単語は
  「アプリの単語帳に追加しておくといいですよ: <単語> / <意味>」と添える。
- 口調: 簡潔・前向き・ただし甘やかさない。ハードワークを支えるコーチであること。
```

`{START_DATE}` はアプリを使い始めた日(アプリのDay表示から逆算可)に置き換える。

## ボットがURLを読めない場合

配信サービス(GAS + LINE Messaging API 等)側で curriculum.json を取得してメッセージを組み立てる。
GASなら `UrlFetchApp.fetch(...)` + 時間主導トリガーで上記フローを実装できる。
ボットの実装方式が確定したら、この節に具体コードを追記すること。
